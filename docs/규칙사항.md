# Git Convention
|구분|로그|
|:-:|:-:|
|기능 구현|feat|
|(오류) 수정|fix|
|삭제|delete|
|리펙터링|refactor|
|테스트|test|
|문서화|docs|
|긴급 수정|hotfix|
  
> branch는 위를 따라 **구분/이슈번호/세부설명(선택)** 형식으로 작성한다.
- 예시 1: feat/#11
- 예시 2: test/#15/create-user


> develop으로 push/merge 권한은 각 파트 리더에게,
  main으로 push/merge 권한은 총괄 리더에게 있다.(hotfix는 예외)
